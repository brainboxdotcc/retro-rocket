Detailed Description

Interfaces for accessing, converting, and formatting real-time clock (RTC) and timezone values.

Author
Craig Edwards (craig.nosp@m.edwa.nosp@m.rds@b.nosp@m.rain.nosp@m.box.c.nosp@m.c)
Copyright
Copyright (c) 2012–2025

This header defines types and functions for:

  • Reading the hardware Real-Time Clock (RTC) via CMOS I/O ports
  • Representing dates/times as POSIX time_t values and datetime_t components
  • Loading and interpreting IANA tzdb time zone files (TZif format)
  • Converting between UTC and local time with DST adjustments
  • Generating formatted date/time strings for display

typedef int64_t time_t

POSIX-style time representation.

Number of seconds since 00:00:00 UTC on 1 January 1970 (the epoch). This implementation uses a signed 64-bit type to allow for historical and far-future date handling beyond the 2038 limit.

ACPI Root System Description Pointer (RSDP)

Format of an ARP packet (per RFC 826)

Register FIS: Host to Device (H2D).

Root System Description Table (RSDT)

Generic ACPI System Description Table Header.

This structure serves as the entry point to the ACPI system description tables. Its location is typically found by scanning the BIOS memory range. It may point to either the RSDT (ACPI 1.0) or XSDT (ACPI 2.0+).

All ACPI tables begin with this header. It is used to identify table type, length, version, and source information.

Points to other system description tables using 32-bit addresses. Superseded by XSDT in ACPI 2.0+ which uses 64-bit addresses.
