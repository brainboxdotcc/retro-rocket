Find the first contiguous run of FREE (0) bits of at least a given length.

Searches within an L0 bitmap sector for a run of free sectors starting from a specified bit offset. Returns the local bit index of the first sector in the run, or UINT64_MAX if no suitable run is found.

Parameters

offsetLocal bit index within the group to begin the search.

Returns
Local bit index of the first sector in the run, or UINT64_MAX if not found.

Count contiguous FREE (0) bits at the start and end of an L1 group's L0 bitmap sector.

Scans the on-disk bitmap for a given L1 group to determine:

  • The length of the leading run of free sectors (prefix).
  • The length of the trailing run of free sectors (suffix).

These values are useful for quickly evaluating whether an extent may span multiple L1 groups without fragmentation.

Parameters
bufPointer to the L0 bitmap sector buffer for the group.
group_bitsNumber of valid bits in the group (may be less than RFS_L1_GROUP_SECTORS for the last group).
out_prefixOutput pointer for the number of free bits from the start.
out_suffixOutput pointer for the number of free bits from the end.

uint64_t  g

inlinestatic

Update L2 cache state for a given L1 group.

This function marks the free/used status of a specific L1 group in the higher-level L2 bitsets. It is used whenever an extent changes state (allocated or freed) so that the hierarchical free space map remains consistent.

The RFS free space tracking hierarchy has three levels:

  • L0: On-disk bitmap, one bit per sector, stored in the free space map.
  • L1: In-RAM summary of each group of RFS_L1_GROUP_SECTORS sectors (4096 sectors = 2 MB at 512 B/sector). Each group tracks:
    • l1_free_count[g] - number of free sectors in the group.
    • l1_not_full bit - set if the group has any free space.
    • l1_all_free bit - set if the group is entirely free.
  • L2: In-RAM summary of super-groups, each covering RFS_L2_GROUPS_PER_SUPER L1 groups. L2 bitsets indicate:
    • l2_not_full - at least one L1 group inside is not full.
    • l2_all_free - all L1 groups inside are fully free.

This function recalculates the L2 state for the super-group containing the specified L1 group index g. It must be called whenever l1_free_count[g] changes due to marking or unmarking space.

Parameters
infoPointer to the mounted RFS volume structure.
gIndex of the L1 group whose parent L2 super-group should be updated.
Note
  • Assumes RFS_L1_GROUP_SECTORS is exactly 4096.
  • Does not update the on-disk L0 bitmap - that is handled separately by the allocation/free functions.
  • This is an inline helper and is not intended for direct external use.
