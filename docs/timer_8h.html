Detailed Description

Timer and delay functions (APIC-driven, 100 Hz tick rate).

Timers are based on the Local APIC timer, running at a fixed resolution of 1/100th of a second (10 ms). A global tick counter is incremented on each timer interrupt, and can be queried via get_ticks(). Sleep functions busy-wait or HLT until the tick counter advances.

The PIT (8253/8254) is also configured to support PC speaker tone generation for beep().

void beep ( uint32_t  pitch)

Generate a short PC speaker beep at the given pitch.

The tone automatically stops after ~0.5 seconds, controlled by the timer interrupt.

Note
This only works if PIT channel 2 is wired to a physical speaker, which is not the case on many modern systems. Reliable only in QEMU/Bochs and older PCs.

Timer interrupt callback.

Invoked by the Local APIC (or PIT fallback) at 100 Hz. Increments the global tick counter, executes registered idle timer callbacks, auto-flips the framebuffer if enabled, and handles PC speaker timeouts.

This should not normally be called by user code.

Parameters
isrInterrupt vector number.
errorcodeError code pushed by CPU (if any).
irqIRQ number (APIC routing).
opaqueOpaque pointer for ISR framework use.

This is triggered either by the Local APIC timer interrupt, or by the PIT.
