Root System Description Table (RSDT)

Generic ACPI System Description Table Header.

This structure serves as the entry point to the ACPI system description tables. Its location is typically found by scanning the BIOS memory range. It may point to either the RSDT (ACPI 1.0) or XSDT (ACPI 2.0+).

All ACPI tables begin with this header. It is used to identify table type, length, version, and source information.

Points to other system description tables using 32-bit addresses. Superseded by XSDT in ACPI 2.0+ which uses 64-bit addresses.

Find a partition of a given type on a device.

Given a device name and partition type ID, searches for a matching partition and fills the provided output parameters with information about it.

Parameters
device_nameName of the device to search.
partition_typeMBR partition type ID to look for.
found_guidBuffer to store the GUID of the found partition (if GPT).
partition_type_guidGUID string of the partition type to match (if GPT).
partition_idFilled with the MBR partition index, or 0xFF if GPT.
startFilled with the start LBA if found.
lengthFilled with the length in sectors if found.
Returns
true if partition found and output parameters are filled.
false if no matching partition was found.
