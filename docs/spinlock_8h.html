Detailed Description

Simple x86 spinlock implementation for Retro Rocket kernel.

Author
Craig Edwards

Provides lightweight synchronisation primitives for protecting short critical sections between multiple CPUs or interrupt contexts. Designed for cases where blocking or sleeping is not possible.

Note
Spinlocks are busy-waiting and should only be used to protect fast operations (e.g. allocator, PID table, queue updates). Long critical sections will waste CPU cycles.

spinlock_t

Opaque 32-bit integer used as a spinlock state.

A value of 0 indicates the lock is free; non-zero indicates held.
